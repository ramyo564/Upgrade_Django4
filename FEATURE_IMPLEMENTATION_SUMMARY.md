# 기능 구현 정리 (Version 1 중심)

이 문서는 현재 코드 기준으로 구현된 기능을 정리하고, 각 기능의 주요 목적을 `사용자 경험(UX) 개선`, `기능 개선`, `최적화`, `보안/안정성` 관점으로 분류한 문서입니다.

## 목적 분류 기준

- `UX 개선`: 사용자가 더 빠르고 편하게 사용하도록 만드는 개선
- `기능 개선`: 비즈니스 요구를 충족하는 핵심 기능 추가/강화
- `최적화`: 조회/탐색/처리 효율을 높이는 개선
- `보안/안정성`: 계정·관리자·주문 데이터의 신뢰성과 안전성 강화

## 기능 구현 목록

| 도메인 | 기능 | 구현 요약 | 주요 목적 |
| --- | --- | --- | --- |
| 계정 | 회원가입 + 이메일 인증 활성화 | 가입 후 이메일 토큰 링크 검증이 완료되어야 계정 활성화 | 보안/안정성 |
| 계정 | 로그인/로그아웃 | 커스텀 유저(Account) 기반 인증 처리 및 로그인 상태 관리 | 기능 개선 |
| 계정 | 비밀번호 찾기/재설정 | 이메일 토큰 검증 후 비밀번호 재설정 | 보안/안정성 |
| 계정 | 프로필 관리 | 프로필 이미지/연락처/주소 등 사용자 정보 수정 | UX 개선 |
| 계정 | 마이페이지 주문 조회 | 내 주문 목록/상세(주문번호 기준) 확인 | UX 개선 |
| 상품 | 카테고리/상품 상세 페이지 | 카테고리별 상품 진입, 상품 상세/갤러리 표시 | 기능 개선 |
| 상품 | 옵션(색상/사이즈) 선택 | Variation 모델로 옵션 관리, 상세/장바구니에 반영 | 기능 개선 |
| 장바구니 | 장바구니 추가/수량 증감/삭제 | 동일 상품+옵션 조합 시 수량 증가, 아니면 신규 라인 추가 | 기능 개선 |
| 장바구니 | 비회원 세션 장바구니 유지 | 세션 cart_id로 비로그인 상태에서도 장바구니 유지 | UX 개선 |
| 장바구니 | 로그인 시 장바구니 병합 | 세션 장바구니와 회원 장바구니를 옵션 단위로 병합 | UX 개선 |
| 주문 | 주문서 생성/주문번호 발급 | 배송지 정보 저장, 세금 계산, 주문번호 생성 | 기능 개선 |
| 결제 | PayPal + KakaoPay 이중 결제 | SDK(PayPal) + REST API(카카오페이) 방식 동시 지원 | 기능 개선 |
| 결제/주문 | 결제 완료 후 주문 확정 처리 | 결제내역 저장, OrderProduct 생성, 옵션 이관, 재고 차감, 장바구니 비움 | 보안/안정성 |
| 결제/주문 | 주문 완료 화면/이메일 알림 | 주문 완료 페이지 렌더링 및 주문 확인 메일 발송 | UX 개선 |
| 리뷰 | 리뷰(댓글) 작성/수정 | 1인 1상품 리뷰 업데이트(있으면 수정, 없으면 등록) | 기능 개선 |
| 리뷰 | 구매자/로그인 사용자만 리뷰 허용 | 비로그인 또는 미구매 사용자의 리뷰 작성 제한 | 보안/안정성 |
| 리뷰 | 평균 별점 + 리뷰 수 집계 | `Avg`, `Count` 집계로 상품 평점/리뷰 개수 계산 | 기능 개선 |
| 리뷰 UI | 별점 시각화(반별 포함) | 평균 평점 및 리뷰 평점을 별/반별 아이콘으로 표시 | UX 개선 |
| 탐색 | 검색 기능 + 검색 결과 개수 | 상품명/설명 키워드 검색, 결과 수 노출 | UX 개선 |
| 탐색 | 페이지네이션 | 상품 목록을 페이지 단위로 분할 조회 | 최적화 |
| 탐색 | 정렬/필터(가격/신상품/평점/옵션) | 가격·신상품·평균평점 정렬 + 색상/사이즈 필터 적용 | UX 개선 |
| 주문 UX | 주소 자동입력(Daum Postcode) | 우편번호/도로명 주소 검색으로 입력 부담 감소 | UX 개선 |
| 공통 UI | 헤더 장바구니 카운터 | context processor로 전역 장바구니 수량 표시 | UX 개선 |
| 보안 | 관리자 보호(허니팟 + 별도 관리자 경로) | 가짜 admin 엔드포인트 + 실제 관리자 경로 분리(`securelogin`) | 보안/안정성 |

## 참고

- 위 항목은 주로 `accounts`, `store`, `carts`, `orders`, `templates` 앱/파일의 현재 구현을 기준으로 정리했습니다.
- API 전환(Version 2) 관련으로 `accounts/views_api.py` 기반 회원가입/로그인(JWT) 엔드포인트도 일부 구현되어 있습니다.
